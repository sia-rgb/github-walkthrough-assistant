# GitHub 项目 Walkthrough 助手

1. 项目目标
帮助用户快速理解 GitHub 上的开源项目项目，包括 README、核心脚本，通过中文翻译、术语注释和按需大白话辅助说明，生成结构化项目概览。

2. 项目价值
解决技术文档难以理解、术语晦涩、项目逻辑复杂或日志信息繁杂的问题。用户无需完全依赖外语能力或专业背景，即可：
- 获取中文翻译的 README 内容
- 理解专业术语及其背景信息
- 对复杂或抽象概念获得可选大白话辅助说明
- 在进一步分析后，获取项目核心功能、核心技术和运行训练需求的结构化概览

3. 输入
- GitHub 仓库 URL
- 可选：项目文件或日志段落粘贴

4. 输出
- README.md 中文专业翻译
- 专业术语注释和关键词解释
- 按需生成大白话辅助说明
- 项目概览四点：
  1. 项目目标
  2. 解决的问题（区分企业级 vs 个人开源项目）
  3. 核心技术/功能
  4. 硬件/软件要求

5. 核心流程
GitHub URL → 仓库API抓取 → DeepSeek / LLM 处理
→ 中文翻译 + 术语注释 → 按需大白话辅助说明 → 项目概览四点 → 整合输出

6. 相关定义

### 核心技术 / 功能
- 定义：项目实现所必须的技术方案、算法、架构或功能模块，是抽象层面的技术要求。
- 举例：
  - Transformer 架构
  - 混合注意力机制
  - 多 Agent 协作
- 特点：
  - 抽象层面，体现“做什么”和“用什么技术实现”
  - 强调技术思路和架构设计，而不是运行依赖
  - 帮助理解项目技术流派或设计模式

### 硬件 / 软件要求
- 定义：项目运行或训练所必须满足的客观资源约束，包括计算硬件和软件环境。
- 举例：
  - GPU 型号与数量（如 NVIDIA A100, 1-4 张）
  - CPU 核数、内存大小
  - 依赖库
- 特点：
  - 具体资源层面，体现“能否运行”和“依赖环境是否满足”
  - 包括依赖库，但不等同于核心技术/功能
  - 帮助评估项目可执行性和硬件需求
