PROJECT_PLAN.md

1. 项目目标
帮助用户快速理解 GitHub 上的开源项目或企业项目，包括 README、核心脚本，通过中文翻译、术语注释和按需大白话辅助说明，生成结构化项目概览。

2. MVP 路线

MVP 1: README 中文翻译 + 注释
目标: 抓取项目 README，生成中文翻译，并标注专业术语和关键词。
输入: GitHub 仓库 URL
预期输出:
- 中文 README
- 专业术语注释
核心难点:
- 中英文混杂内容翻译准确性
- 术语解释上下文理解
验收标准:
- 翻译中文流畅、专业
- 术语注释清晰、可理解
核心技术:
- DeepSeek / LLM
暂不做:
- 脚本/Agent 分析
- 前端交互展示




MVP 2: 项目概览生成
目标: 提炼项目核心信息，包括目标、解决的问题、核心技术/功能和运行训练需求。
输入: GitHub 仓库的所有文件
预期输出:
- 项目目标
- 解决的问题（区分企业级 vs 个人开源项目，依据发布者类型 User / Organization）
- 核心技术/功能 (Core Technologies & Functions)
- 硬件/软件要求 (Hardware & Software Requirements)
核心难点:
- 自动判断企业级或个人项目
- 提炼核心技术/功能并可理解
- 明确区分核心技术/功能与硬件/软件 requirements
验收标准:
- 概览四点清晰完整
- 企业级 vs 个人项目描述合理
核心技术:
- DeepSeek / LLM 指令模板
暂不做:
- 前端交互展示



MVP 3: 按需大白话辅助说明
目标: 对复杂或抽象概念生成可选大白话辅助说明。
输入: 用户选中或鼠标高亮的项目段落/代码块
预期输出:
- 按需生成大白话辅助说明，保持专业内容不变
- 显示在原段落下方或折叠区域
核心难点:
- 用户选中内容的识别
- 异步请求与性能优化
- 输出整合与展示
验收标准:
- 大白话仅作为辅助，不替代专业内容
- 用户可随时触发，长文档不会被淹没
核心技术:
- DeepSeek / LLM
暂不做:
- 全局自动大白话生成
- 高度可视化图形展示




MVP 4: 网页交互界面 (UI Web)
目标: 将 MVP1~MVP3 的功能整合到一个网页交互界面，可部署到公网。
输入: GitHub 仓库 URL
预期输出:
- 左侧主文档区显示 README 中文翻译及术语注释
- 单个按钮触发大白话辅助说明（用户选中段落后点击生成）
- 右侧显示项目概览四块（项目目标、解决问题、核心技术/功能、硬件/软件要求）
- 布局清晰、直观，用户可快速浏览文档及概览
核心难点:
- 页面布局设计，使左侧文档区与右侧概览区信息可同时可见
- 用户选中段落触发大白话的交互逻辑
- 保证大白话输出整合不干扰主文档
- 兼顾长文档滚动及可折叠显示
验收标准:
- 用户打开网页即可看到 README 中文翻译及项目概览
- 用户选中段落点击按钮即可生成大白话
- 页面布局合理，信息展示直观
核心技术:
- 前端网页设计（布局、滚动、折叠显示）
- 后端与 DeepSeek / LLM 的接口交互
暂不做:
- 高度复杂的可视化图表
- 全局自动大白话生成
